🎮 一、互動與使用體驗提升
1️⃣ 自動計算與即時顯示

當使用者輸入 constant 與 score 時，自動更新 playptt，而不用按「click me to get playptt」。 O

可以用 input 的 oninput 事件即時觸發函數。 (沒用)

2️⃣ 動態顯示 B30 平均 O

讓頁面根據表格自動計算 B30 平均值，而不是寫死在 HTML。

用 JS 掃描所有 .playptt 欄位，自動求平均，這樣未來新增歌曲不用手動改。

3️⃣ 可排序表格 O

點擊表頭可依照「常數」、「分數」、「ptt」升/降排序。

可以用純 JS 或現成的輕量套件（例如 sorttable.js
）。

4️⃣ 搜尋/篩選功能

加一個搜尋框輸入歌曲名稱，即時篩選表格列。 O

或新增「難度篩選（FTR/ETR/BYD）」下拉選單。

🎨 二、視覺與資訊呈現
5️⃣ 加入圖表分析

用 Chart.js 或 ECharts 將 B30 分布畫成長條圖或折線圖。
例如顯示「常數 vs ptt」的關係，或顯示你哪一年進步最多。

6️⃣ 成就與里程碑時間軸(沒用)

你已經在表格內有 timeline1, timeline2...
可以把它們抽出來變成一條 時間軸 (timeline)，例如：

2022.3/15 — 開始玩
2022.6/19 — ptt10.00
2023.2/8 — ptt11.00
2024.10/13 — ptt12.00

用 CSS 動畫讓每個節點會亮起或滑動顯示。

7️⃣ 夜間模式 / 主題切換(沒用)

加個按鈕切換「亮/暗模式」，調整背景、字體顏色。

可用 CSS 的 :root 變數加 JS toggle。

💾 三、資料與擴展性
8️⃣ 讀取外部 JSON 檔

把歌曲資料（名稱、難度、分數、常數、ptt、圖片）放進 JSON 檔，用 JS 讀入自動生成表格。
這樣維護更容易，也方便未來擴充成百首歌。

9️⃣ 使用者成績上傳/儲存

若你想讓頁面變成「小工具」而不只是展示頁，可以用 localStorage 儲存輸入的曲目與分數。

使用者重新開啟網頁時能看到自己上次的紀錄。

🔟 匯出功能

加一個「匯出 CSV」或「生成個人成績卡」的按鈕，
將表格內容轉成檔案下載，或生成 PDF 檔。

🌐 四、延伸玩法
🧠 排行榜或個人進步追蹤

若你之後想連上伺服器，可以讓使用者登入並上傳他們的 B30，
然後自動生成「排行榜」或「歷史進步曲線」。

🎵 音樂預覽連結

每首歌旁邊加一個按鈕連到 YouTube/Arcaea OST，讓人可以直接聽。

🎲 隨機推薦強化版

目前隨機推薦是單純亂選圖片，可以改成：

依難度或未 FC 曲目加權抽選。

加入動畫（圖片旋轉、閃爍再停下）。
